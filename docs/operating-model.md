# Operating Model

## Ensemble Core

Ouroboros is not one trigger engine.

It is a coordinated stack of distinct execution styles:

- trend continuation
- breakout
- mean reversion
- structure-aware price action
- regime-aware adaptation

## Coordination Principle

Oracle exists to:

- resolve conflicts
- cap overlap
- shape portfolio exposure
- turn independent signals into a coherent execution posture

## Product Boundary

Ouroboros is the flagship ensemble.

Standalone products like Indian King Cobra and Terciopelo belong to the same ecosystem, but not to the core Ouroboros execution stack.
